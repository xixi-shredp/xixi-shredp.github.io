操作系统 = 对象 + API

操作系统的启动

  1. Firmware 阶段：在CPU reset后，加载操作系统
  2. 操作系统初始化阶段：
    • 扫描系统中的硬件、初始化数据结构……
    • 加载第一个进程 (状态机)init，操作系统启动
  3. 操作系统执行阶段
    • 状态机在 CPU 上执行
    • 允许执行 syscall 进入操作系统代码

Linux操作系统启动之后的阶段

  1. 初级阶段
    1. 加载剩余必要的驱动程序，例如磁盘/网卡
    2. 挂载必要的文件系统
    3. 将根文件系统和控制权移交给另一个程序，例如 systemd
      switch_root 背后的系统调用：pivot_root
  2. 第二级阶段：
    • systemd 通过 mount 系统调用配置文件系统（通过/etc/fstab）

操作系统对象

  • 文件：有 “名字” 的对象
  • 字节流 (终端，管道) 或字节序列 (普通文件)
  • 文件描述符: 指向操作系统对象的 “指针”
    对象的访问都需要指针

操作系统API（系统调用）

syscall指令: “跳转并获得无限的权力”

syscall之后：
可以直接访问 I/O 设备和内核数据结构

进程管理API

  • fork
  • execve
  • exit

内存管理API

  • mmap
  • munmap
  • mprotect

文件描述符API

  • open
  • close
  • read/write (解引用)
  • lseek (指针内赋值/运算)
  • dup (指针间赋值)

管道

  • pipe（匿名管道）
  • mkfifo（命名管道）

返回两个文件描述符，进程同时拥有读口和写口:

  • 读口在没有数据时会等待，如果读者关闭，会收到 SIGPIPE 信号 Broken Pipe
  • 写口：有读者打开时，会写入操作系统的缓冲区并返回
  • 具有原子性：
    write 如果 “不太多”，一对 write-read 是原子的，write 如果超过 PIPE_BUF，可能会被拆成多份

系统调用之上：C 标准库

语言机制上的运行库:
大部分可以用 C 语言本身实现，少部分需要一些 “底层支持”

标准化:（稳定，可靠，移植性）

  • ISO IEC 标准的一部分
  • POSIX C Library 的子集

基础编程机制的抽象

基础头文件

stdint.h
stdbool.h
float.h
limits.h
stdarg.h

string.h

系统调用与环境的抽象

Standard I/O: stdio.h

  • FILE*：背后其实是一个文件描述符
  • 封装了文件描述符上的系统调用 (fseek, fgetpos, ftell, feof, …)
  • printf() family
  • popen 和 pclose
  • err, error, perror

环境变量:
envp：execve() 传递给进程的 “环境”

全局变量 environ
操作系统有一个 “初始状态” Initial Process Stack ,
libc 调用 main 前还会继续初始化这个 “初始状态”

操作系统外壳：Shell

基本功能:: 启动系统中的应用程序

Command-line Interface (CLI)

UNIX Shell:: “把用户指令翻译成系统调用” 的编程语言
基于文本替换的快速工作流搭建

Graphical Shell (GUI)
