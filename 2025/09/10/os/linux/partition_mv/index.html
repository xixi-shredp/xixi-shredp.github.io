Linux 下的分区迁移

普通分区迁移

  1. free space -> 新建ext4分区
  2. 分区数据拷贝（按字节复制）
  1. 检查分区完整性
  1. 修改文件系统大小

dd按字节复制，会把文件系统的所有信息都复制到新分区，因此新分区的大小显示依旧是原有分区大小。

  1. 在新的分区的/etc/fstab内修改UUID：

grub引导时按照uuid寻找根目录，而dd按字节拷贝后的分区与根分区具有完全相同的uuid，grub引导时会出现错误。

设置随机uuid

或

  1. 设置自动挂载：修改/etc/fstab

重启，验证。

对于根分区

  1. 修改grub，重建引导（需要在 Live CD或者 Live USB 上进行）

boot-repair修改默认os-boot分区时，不能在当前os运行时修改

在 Live CD / Live USB 上下载boot-repair：（先配好网络）

启动boot-repair，选择Advanced Options，修改os-default-boot到目标分区/dev/sda1，重启，验证。

对于 swap 分区

  1. 将要移动到的目标分区格式化为 swap 分区格式
  • 图形界面：Disk 直接 edit partition
  • 命令行：fdisk 创建 swap 分区
  1. mkswap /dev/sda 创建 swap
  2. 将新分区的 UUID 更新到 /etc/fstab 中的 swap 项

分区表 Partition Table
分区表 Partition Table目前常用的分区表有 MBR 分区表和 GPT 分区表 MBR 分区表 (Master Boot Record) 存储在硬盘的第一个扇区上(分区表的大小是固定的) 兼容性比较好 最多只能识别2TB的空间 最多只能支持4个主分区或3个主分区+1个扩展分区(逻辑分区不限制) GPT 分区表 (GUID Partition Table) 支持2TB以上的大硬盘 在Windows系统下可以支持128个主分区 在GPT分区表头中可自定义分区数量的最大值(GPT分区表的大小不是固定的)
